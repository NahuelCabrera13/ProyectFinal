# Project Charter

**Proyecto:** IAStock  
**Fecha:** 21/01/2026  
**Versión:** 1.0

----

## 1. Objetivos del Proyecto

### Propósito
Desarrollar una aplicación de control de inventario con asistencia de inteligencia artificial que permita a negocios de cualquier tamaño gestionar su mercancía de forma eficiente, automatizar tareas repetitivas y tomar decisiones basadas en datos.

### Objetivos SMART

| # | Objetivo |
|---|----------|
| 1 | Crear una interfaz intuitiva que permita gestionar inventario en tiempo real durante las primeras 6 semanas de desarrollo |
| 2 | Implementar un asistente IA funcional que responda consultas del usuario y automatice tareas básicas para la entrega del MVP |
| 3 | Desarrollar un sistema de predicción de stock que alerte sobre necesidades de reabastecimiento con al menos 80% de precisión |

## 3. Alcance del Proyecto

### In-Scope (Incluido)
- Gestión básica de inventario (agregar, editar, eliminar productos)
- Asistente IA integrado para consultas y automatización
- Sistema de predicción de demanda y alertas de stock bajo
- Dashboard con visualización del estado del inventario
- Funcionalidad para agendar citas y entregas

### Out-of-Scope (Excluido)
- Integración con sistemas de contabilidad externos
- Aplicación móvil nativa (solo versión web para MVP)
- Funcionalidades avanzadas de facturación
- Soporte multi-idioma (solo español para MVP)
- Sistema de pagos integrado

## 4. Riesgos y Mitigación

| Riesgo | Probabilidad | Mitigación |
|--------|--------------|------------|
| Dependencia de APIs de IA externas (costos, disponibilidad) | 40% | Investigar alternativas y establecer límites de uso; considerar modelos locales como respaldo |
| Falta de experiencia con tecnologías de IA | 60% | Asignar tiempo de aprendizaje en etapas tempranas; consultar documentación y tutoriales |
| Dificultad para coordinar reuniones del equipo debido a horarios laborales y compromisos personales de los integrantes | 75% | Establecer comunicación asíncrona mediante herramientas digitales (Discord, WhatsApp); programar reuniones breves en horarios flexibles; dividir tareas de forma independiente para avanzar sin depender de reuniones presenciales |
| Tiempo insuficiente para completar todas las funcionalidades del MVP debido a la carga académica y laboral del equipo | 55% | Priorizar las funcionalidades esenciales; aplicar metodología ágil con entregas incrementales; definir un MVP mínimo viable si el tiempo no alcanza |
